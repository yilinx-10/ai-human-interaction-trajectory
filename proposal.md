Project Proposal - DATA 31500

Motivation
Our research question is: to what extent do users’ topic choices, methodologies, and belief systems exhibit innovation relative to their own prior trajectory during LLM usage? AI assistants like ChatGPT and Claude have become commonly used tools in daily work. Recently, major AI platforms started supporting full conversation history export, so users can now access structured records of their long-term interactions with AI. However, current AI platforms have obvious limitations: cross-conversation memory is very limited, and conversations are isolated from each other. An active user may accumulate hundreds of conversations over a few months, but each time they start a new conversation, the context from previous ones is almost entirely lost.  Some existing visualization tools can display conversation history, but they treat conversations as flat, disconnected artifacts. They do not surface the underlying structure of users' thinking, nor do they offer the kind of layered, relational exploration that makes patterns across sessions visible and actionable. Our work addresses this gap by modeling a user's intellectual history as an evolving belief network, navigable through a multi-layered interactive interface that lets users move fluidly between the level of individual conversations, thematic clusters, and overarching reasoning trajectories.
Some users report difficulty recalling how they solved similar problems in previous projects, particularly when prior interactions are distributed across multiple sessions.  Valuable ideas, reasoning chains, and workflows get buried and gradually forgotten. In addition, longitudinal engagement with AI systems may shape users’ reasoning practices or problem-framing strategies. Yet these shifts are not always transparent: users may lack tools to reflect on how their thinking has changed over time, to distinguish between self-generated and AI-influenced ideas, or to identify potential inconsistencies in their reasoning.
We frame LLM usage as a potential source of innovation or repetition, and adopt a network perspective. A user’s intellectual trajectory can be modeled as networks, where nodes represent beliefs, topics,  and methods, and edges represent logical relationships between nodes such as causal, supportive, or contradictory connections. Innovation can be identified in three ways:
Node addition: the user introduces an entirely new topic or belief
Edge addition: the user establishes a new connection between existing topics
Network rewiring: the user’s reasoning pattern changes, for example shifting from “A causes B” to “A and B are mutually reinforcing”
Our within-user baseline approach tracks each user’s own intellectual evolution rather than comparing across users.
This tool is designed for all users who have accumulated AI conversation history, including managers handling multiple projects, researchers, students, and anyone who wants to understand their AI interaction patterns. We plan to build an interactive visualization tool where users upload exported AI conversation data, the system extracts topics, methods, and reasoning patterns to construct a belief network, and presents it through multi-layered interactive visualizations. The tool has two main goals:
Recall: facilitate  finding relevant past conversations and workflows, and 
Discovery: revealing hidden connections, identifying unresolved contradictions, and tracking innovation patterns. 
The tool also supports multi-user network comparison for team collaboration. The final deliverable will be a deployed prototype using team members’ own conversation data.
How will visualization, web technology, and/or AI help?
AI: After users upload conversation data, the system first uses Model Context Protocol to unify export formats from different platforms into a standardized internal representation. Then sentence embedding models encode each conversation as a vector for semantic similarity computation and clustering. Next, LLM analyzes each conversation: extracting topic labels and method keywords, inferring belief structures including claims, assumptions, and conclusions as nodes, and their logical relationships including causal, supportive, and contradictory connections as edges. This transforms scattered conversations into a structured belief network.
Visualization: We design three switchable layers, each showing the user’s intellectual trajectory from a different angle. The Topic layer (What) uses an embedding-based scatter plot where similar conversations cluster together, with color encoding topic and size encoding importance. The Method layer (How) displays methods and workflows over a timeline, showing methodology evolution. The Belief/Reasoning layer (Why) uses a force-directed network to show claims, assumptions, and conclusions with edge styles distinguishing causal, supportive, and contradictory relationships. These three layers answer what the user thinks about, how they approach problems, and why they reason the way they do.  When starting a new project, the tool highlights relevant areas in the past belief network, enabling cross-project knowledge transfer. 
Web Technology: Svelte builds the frontend interface, and D3 handles visualization rendering and interaction including force layout, brush filtering, and zoom/pan. Core interactions include switching between layers, time range filtering, clicking nodes for detail views, and comparing network changes across time periods. Users can drag past nodes onto new project branches to explore connections. The tool supports multi-user views for team topic overlap analysis. All data processing runs locally in the browser for privacy.

Potential challenges and responses:
LLM extraction of beliefs may not be accurate enough; we can try starting with Topic and Method layers first, and try different LLMs to compare quality. Some conversations are very short and lack context for meaningful belief extraction; we can try labeling them as “unexpanded” nodes. Conversations may contain multiple topics; we can try segmenting by turn-taking structure and semantic transitions. The same belief may appear across conversations with different wording; we can try using embedding similarity thresholds to merge similar nodes.

We want to be able to deplopy it as a web extension.
